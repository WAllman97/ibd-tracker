# MVP Definition

The MVP should only do 3 things:

1. Allow users to enter daily symptoms
2. Save the data
3. Display useful trends

Everything else comes later.

---

# Key Principle

Avoid overbuilding early.

Focus on:
- consistency
- usability
- speed
- simplicity

A small working app is more valuable than a huge unfinished one.
